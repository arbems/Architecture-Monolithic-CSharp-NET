WORKFLOWS (Optional)


* Responsible for coordinating all the elements of the application.

* This layer coordinates transactions and data persistence, data validations, 
security aspects with authentication.

* This layer controls and manipulates the domain. This layer will be responsible for calling the repositories of the domain
to obtain the information, instantiate the services of the domain and other necessary components, and finally persist the changes in our database.

* In this layer, we would also create interfaces and implementations for services, DTOs etc, if necessary.
